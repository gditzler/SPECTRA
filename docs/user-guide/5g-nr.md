# 5G NR Waveforms

SPECTRA provides 5G New Radio (NR) primitive waveforms following the
**3GPP TS 38.211** specification. The classes are baseband generators with
configurable numerology, and they are suitable for training spectrum
classifiers, populating wideband scenes alongside legacy emitters, or
producing reference signals for receiver testing.

All NR classes inherit `Waveform` and are deterministic given a seed
(safe across DataLoader workers).

---

## Available classes

| Class | Purpose |
|-------|---------|
| `NR_OFDM` | Generic 5G NR OFDM waveform with configurable numerology and resource blocks |
| `NR_SSB` | 5G NR Synchronization Signal Block — 240-subcarrier x 4-symbol grid with PSS, SSS, and PBCH data |
| `NR_PDSCH` | 5G NR Physical Downlink Shared Channel with DMRS symbols |
| `NR_PUSCH` | 5G NR Physical Uplink Shared Channel with optional DFT precoding (SC-FDMA) |
| `NR_PRACH` | 5G NR Physical Random Access Channel using Zadoff-Chu preamble sequences |

---

## Quickstart

```python
from spectra.waveforms import NR_SSB

# Defaults: numerology mu=1 (30 kHz SCS), cell identity group 0, sector 0
ssb = NR_SSB()
iq = ssb.generate(num_symbols=4, sample_rate=30.72e6, seed=0)
print(iq.shape, iq.dtype)   # (8784,) complex64
```

The `generate()` call follows the standard `Waveform` interface:
`num_symbols` is the number of **slots** (or SSB blocks for `NR_SSB` and
preambles for `NR_PRACH`) to produce. `sample_rate` is used to compute
time-domain spacing; `seed` makes the output reproducible.

---

## Configuring numerology

NR numerology (denoted μ) selects the subcarrier spacing, which in turn
determines slot duration and occupied bandwidth per resource block.
`NR_OFDM`, `NR_PDSCH`, and `NR_PUSCH` all accept a `numerology` parameter:

| μ | Subcarrier spacing | Symbols per slot | Typical use |
|---|--------------------|-----------------|-------------|
| 0 | 15 kHz | 14 | LTE-compatible, FR1 |
| 1 | 30 kHz | 14 | FR1 sub-6 GHz (default) |
| 2 | 60 kHz | 14 | FR1 / FR2 |
| 3 | 120 kHz | 14 | FR2 mmWave |
| 4 | 240 kHz | 14 | FR2 reference signals |

```python
from spectra.waveforms import NR_OFDM

# 60 kHz subcarrier spacing (mu=2), 25 resource blocks
ofdm_60khz = NR_OFDM(numerology=2)
iq = ofdm_60khz.generate(num_symbols=4, sample_rate=61.44e6, seed=0)
print(ofdm_60khz.bandwidth(61.44e6) / 1e6)   # 18.0 MHz
```

`NR_SSB` also takes `numerology`, but `NR_PRACH` does not — PRACH subcarrier
spacing is derived from the preamble format (1.25 kHz for long formats 0–3,
15 kHz for short formats 4+).

### Preset factories

`NR_OFDM` ships two class-method presets:

```python
# FR1 20 MHz channel: mu=1, 51 RBs, 1024-pt FFT
ofdm_fr1 = NR_OFDM.fr1_20mhz()

# NR_SSB n78 band preset: n_id_1=0, n_id_2=0, mu=1
ssb_n78 = NR_SSB.n78()
```

## NR_OFDM

`NR_OFDM` is the generic NR data-channel OFDM generator. Its resource
grid is filled with random constellation symbols; no pilot or DMRS
structure is imposed (use `NR_PDSCH` or `NR_PUSCH` for those).

```python
from spectra.waveforms import NR_OFDM

# mu=1, 52 RBs (10 MHz channel equivalent), QAM-16 data
ofdm = NR_OFDM(numerology=1, num_resource_blocks=52, modulation="qam16")
iq = ofdm.generate(num_symbols=10, sample_rate=30.72e6, seed=7)
print(f"BW = {ofdm.bandwidth(30.72e6) / 1e6:.1f} MHz")
```

**Key parameters:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `numerology` | `1` | Numerology index μ (0–4) |
| `num_resource_blocks` | `25` | Number of resource blocks (1 RB = 12 subcarriers) |
| `fft_size` | `512` | IFFT size for OFDM modulation |
| `modulation` | `"qpsk"` | Subcarrier modulation: `"qpsk"`, `"qam16"`, `"qam64"`, `"qam256"` |

---

## SSB structure

The Synchronization Signal Block occupies a fixed 240-subcarrier
(20 RB) grid across 4 OFDM symbols. SPECTRA's `NR_SSB` builds the
block according to 3GPP TS 38.211:

- **Symbol 0** — PSS (127 subcarriers, subcarriers 56–182) + QPSK
  fill for remaining PBCH subcarriers
- **Symbol 1** — PBCH data (QPSK, all 240 subcarriers)
- **Symbol 2** — SSS (127 subcarriers, subcarriers 56–182) + QPSK
  fill for remaining PBCH subcarriers
- **Symbol 3** — PBCH data (QPSK, all 240 subcarriers)

The PSS and SSS sequences are generated by the Rust backend
(`generate_nr_pss`, `generate_nr_sss`) using the physical cell
identity parameters `n_id_1` (group, 0–335) and `n_id_2` (sector, 0–2).

```python
from spectra.waveforms import NR_SSB

# Cell identity: N_cell = 3 * n_id_1 + n_id_2 = 3*10 + 1 = 31
ssb = NR_SSB(n_id_1=10, n_id_2=1, numerology=1)
iq = ssb.generate(num_symbols=2, sample_rate=30.72e6, seed=0)
print(f"BW = {ssb.bandwidth(30.72e6) / 1e6:.1f} MHz")   # 7.2 MHz (30 kHz x 240)
```

## PDSCH and PUSCH

### NR_PDSCH

`NR_PDSCH` generates a downlink shared channel slot grid with realistic
DMRS placement. DMRS symbols (type A, symbols 2 and optionally others
within the slot) carry orthogonal reference signals on even subcarriers;
odd subcarriers carry data. All other symbols carry full-density data.

```python
from spectra.waveforms import NR_PDSCH

# 52 RBs, QAM-64, 2 DMRS symbols per slot
pdsch = NR_PDSCH(numerology=1, num_resource_blocks=52, modulation="qam64")
iq = pdsch.generate(num_symbols=4, sample_rate=30.72e6, seed=42)
print(f"BW = {pdsch.bandwidth(30.72e6) / 1e6:.1f} MHz")   # 18.7 MHz
```

**Key parameters:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `numerology` | `1` | Numerology index μ (0–4) |
| `num_resource_blocks` | `25` | Number of resource blocks |
| `modulation` | `"qpsk"` | Subcarrier modulation: `"qpsk"`, `"qam16"`, `"qam64"`, `"qam256"` |
| `num_dmrs_symbols` | `2` | DMRS symbols per slot (placed starting at symbol index 2) |

### NR_PUSCH

`NR_PUSCH` is structurally similar to `NR_PDSCH` (uplink data + DMRS
at symbol 2), with an additional option for **DFT precoding**
(DFT-s-OFDM, also known as SC-FDMA) used in power-limited uplink scenarios.

```python
from spectra.waveforms import NR_PUSCH

# Standard CP-OFDM uplink
pusch_ofdm = NR_PUSCH(numerology=1, num_resource_blocks=25, modulation="qpsk")

# SC-FDMA / DFT-s-OFDM (lower PAPR, used in NR uplink)
pusch_scfdma = NR_PUSCH(transform_precoding=True)

iq = pusch_scfdma.generate(num_symbols=4, sample_rate=30.72e6, seed=0)
```

**Key parameters:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `numerology` | `1` | Numerology index μ (0–4) |
| `num_resource_blocks` | `25` | Number of resource blocks |
| `modulation` | `"qpsk"` | Subcarrier modulation: `"qpsk"`, `"qam16"`, `"qam64"`, `"qam256"` |
| `transform_precoding` | `False` | Apply DFT precoding for SC-FDMA waveform |

## PRACH

`NR_PRACH` generates random-access preambles using Zadoff-Chu (ZC)
sequences, which have the constant-envelope and good autocorrelation
properties required for timing acquisition.

**Preamble formats:** formats 0–3 use the long sequence length
(839 subcarriers, 1.25 kHz SCS); formats 4 and above use the short
sequence length (139 subcarriers, 15 kHz SCS).

```python
from spectra.waveforms import NR_PRACH

# Long preamble, format 0, root sequence u=22, cyclic shift 13
prach = NR_PRACH(preamble_format=0, root_index=22, cyclic_shift=13)
iq = prach.generate(num_symbols=1, sample_rate=30.72e6, seed=0)
print(f"BW = {prach.bandwidth(30.72e6) / 1e3:.1f} kHz")

# Short preamble (format 4+), root sequence u=1
prach_short = NR_PRACH(preamble_format=4, root_index=1)
iq_short = prach_short.generate(num_symbols=2, sample_rate=30.72e6, seed=0)
```

A convenience class method is provided:

```python
prach = NR_PRACH.format_0()   # preamble_format=0, root_index=1, cyclic_shift=0
```

**Key parameters:**

| Parameter | Default | Description |
|-----------|---------|-------------|
| `preamble_format` | `0` | PRACH format (0–3 long, 4+ short) |
| `root_index` | `1` | Zadoff-Chu root sequence index u |
| `cyclic_shift` | `0` | Cyclic shift N_cs applied to ZC sequence |

## Use in a wideband scene

NR waveforms drop into `Composer` alongside any other `Waveform` subclass.
The scene engine calls `bandwidth()` to assign frequency slots, so a
`NR_PDSCH` occupying 9 MHz will coexist cleanly with a narrowband `QPSK`
emitter in the same 20 MHz capture:

```python
from spectra.scene import Composer, SceneConfig
from spectra.waveforms import NR_PDSCH, QPSK

scene = Composer(SceneConfig(
    capture_duration=1e-3,
    capture_bandwidth=20e6,
    sample_rate=30.72e6,
    num_signals=2,
    signal_pool=[NR_PDSCH(), QPSK()],
    snr_range=(10.0, 20.0),
))
iq, descs = scene.generate(seed=0)
print(iq.shape, iq.dtype)           # (30720,) complex64
for d in descs:
    print(d.label, f"{d.snr:.1f} dB")
```

Use `NR_OFDM` when you want a generic NR signal without DMRS structure,
`NR_PDSCH` for a realistic downlink channel, or mix multiple NR classes
with different numerologies in the same scene.
